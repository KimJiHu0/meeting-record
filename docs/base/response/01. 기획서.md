# MeetingFlow 간단 기획서

작성일: 2026-06-19  
문서 목적: F/E, B/E, DB 담당자가 간단한 토이 프로젝트 구조를 바로 잡고 개발을 시작할 수 있도록 핵심 화면, 기능, API, 데이터 모델만 정리한다.

## 1. 프로젝트 한 줄 설명

MeetingFlow는 회의 음성을 녹음하고 AI로 분석해 회의록과 마감 일정을 저장하는 간단한 회의 기록 서비스다.

## 2. 목표 사용자

-   개인 사용자 또는 소규모 프로젝트 참여자
-   회의 후 회의록을 빠르게 정리하고 싶은 사용자
-   담당자와 마감일을 놓치지 않고 관리하고 싶은 사용자

## 3. MVP 목표

-   사용자는 로그인 후 회의를 시작하고 브라우저에서 음성을 녹음한다.
-   회의 종료 후 녹음 파일을 AI로 분석한다.
-   AI는 회의 제목, 요약, 주요 내용, 담당자, 마감일을 추출한다.
-   사용자는 분석 결과 팝업에서 내용을 확인하고 추가 메모를 작성한다.
-   사용자는 최종 회의록을 저장하고 회의 목록, 회의 상세, 일정 목록에서 확인한다.

## 4. 핵심 사용자 흐름

1. 사용자가 회원가입 또는 로그인을 한다.
2. 대시보드에서 `회의 시작` 버튼을 클릭한다.
3. 회의 녹음 화면에서 음성 녹음을 시작한다.
4. 회의가 끝나면 `회의 종료` 버튼을 클릭한다.
5. 녹음 파일을 기반으로 AI 분석을 요청한다.
6. AI 분석 결과 팝업에서 회의 제목, 요약, 주요 내용, 담당자, 마감일을 확인한다.
7. 사용자가 추가 메모를 작성하고 카테고리를 선택한다.
8. 회의록을 저장한다.
9. 저장된 회의록은 회의 목록/상세에서 확인하고, 마감일은 일정 목록에서 확인한다.

## 5. 화면 구성

### 5.1 로그인

-   화면 목적: 기존 사용자가 서비스에 진입한다.
-   주요 UI 요소: 이메일 입력, 비밀번호 입력, 로그인 버튼, 회원가입 이동 링크
-   연결 기능: 로그인, 내 정보 조회

### 5.2 회원가입

-   화면 목적: 신규 사용자가 계정을 생성한다.
-   주요 UI 요소: 이름 입력, 이메일 입력, 비밀번호 입력, 회원가입 버튼, 로그인 이동 링크
-   연결 기능: 회원가입

### 5.3 대시보드

-   화면 목적: 사용자의 회의 현황과 주요 일정을 한눈에 보여준다.
-   주요 UI 요소: 회의 시작 버튼, 총 회의 수, 최근 회의 목록, 예정 마감 일정
-   연결 기능: 회의 생성, 회의 목록 조회, 일정 목록 조회

### 5.4 회의 목록

-   화면 목적: 저장된 회의록을 조회하고 상세 화면으로 이동한다.
-   주요 UI 요소: 검색 입력, 상태 필터, 회의 카드/테이블, 상세 보기 버튼
-   연결 기능: 회의 목록 조회, 회의 상세 조회, 회의 삭제

### 5.5 회의 녹음 화면

-   화면 목적: 회의를 녹음하고 AI 분석을 요청한다.
-   주요 UI 요소: 회의 제목 입력, 회의 시작 버튼, 녹음 중 상태, 녹음 시간, 회의 종료 버튼, 분석 요청 버튼
-   연결 기능: 회의 생성, 녹음 저장, AI 분석 요청

### 5.6 회의 분석 결과 팝업

-   화면 목적: AI 분석 결과를 사용자가 확인하고 최종 저장한다.
-   주요 UI 요소: 추천 회의 제목, 회의 요약, 주요 내용, 담당자, 마감일, 카테고리 선택, 추가 메모 입력, 저장 버튼
-   연결 기능: AI 분석 결과 조회, 회의록 저장

### 5.7 회의 상세

-   화면 목적: 저장된 회의록과 사용자 메모를 조회/수정한다.
-   주요 UI 요소: 회의 제목, 카테고리, 회의 요약, 주요 내용, 담당자, 마감일, 추가 메모, 수정 버튼, 삭제 버튼
-   연결 기능: 회의 상세 조회, 회의록 수정, 회의 삭제

### 5.8 일정 목록

-   화면 목적: 회의 분석에서 추출된 마감일 기반 일정을 확인한다.
-   주요 UI 요소: 날짜 필터, 일정 목록, 담당자, 연결된 회의 링크
-   연결 기능: 일정 목록 조회

### 5.9 마이페이지

-   화면 목적: 사용자 정보와 회의/일정 카테고리를 관리한다.
-   주요 UI 요소: 프로필 사진, 유저 ID, 이름, 이메일, 전화번호, 회사명, 소속, 사용자 정보 수정 폼, 카테고리 목록, 카테고리 추가/수정/삭제 버튼
-   연결 기능: 사용자 정보 조회/수정, 카테고리 관리

## 6. 기능 목록

### 6.1 인증

-   회원가입: 이름, 이메일, 비밀번호로 계정을 생성한다.
-   로그인: 이메일과 비밀번호로 인증한다.
-   내 정보 조회: 현재 로그인한 사용자의 기본 정보를 조회한다.

### 6.2 사용자 프로필

-   사용자 정보 조회: 프로필 사진, 유저 ID, 이름, 이메일, 전화번호, 회사명, 소속을 조회한다.
-   사용자 정보 수정: 이름, 전화번호, 회사명, 소속, 프로필 사진을 수정한다.

### 6.3 카테고리 관리

-   카테고리 목록 조회: 사용자가 만든 회의/일정 카테고리를 조회한다.
-   카테고리 생성: 새 카테고리를 생성한다.
-   카테고리 수정: 카테고리 이름과 색상을 수정한다.
-   카테고리 삭제: 사용하지 않는 카테고리를 삭제한다.

### 6.4 회의 관리

-   회의 생성: 회의 녹음을 시작하기 위한 회의 데이터를 생성한다.
-   회의 녹음: 브라우저 녹음 기능으로 음성 파일을 생성한다.
-   회의 종료: 녹음을 종료하고 녹음 파일 정보를 저장한다.
-   회의 목록 조회: 저장된 회의 목록을 최신순으로 조회한다.
-   회의 상세 조회: 회의록, AI 분석 결과, 사용자 메모를 조회한다.
-   회의록 수정: 제목, 카테고리, 메모, 주요 내용을 수정한다.
-   회의 삭제: 회의와 분석 결과를 삭제한다.

### 6.5 AI 분석

-   STT 변환: 녹음 파일을 텍스트로 변환한다.
-   회의 제목 추천: 회의 내용을 기반으로 제목을 추천한다.
-   회의 요약 생성: 회의 내용을 짧게 요약한다.
-   주요 내용 추출: 회의에서 중요한 논의 내용을 추출한다.
-   담당자 추출: 할 일과 연결된 담당자를 추출한다.
-   마감일 추출: 일정으로 관리할 날짜를 추출한다.

### 6.6 회의록 저장

-   분석 결과 확인: 팝업에서 AI 분석 결과를 확인한다.
-   추가 메모 작성: 사용자가 회의에 대한 보충 메모를 작성한다.
-   카테고리 선택: 회의록에 카테고리를 지정한다.
-   최종 저장: AI 분석 결과와 사용자 메모를 회의록으로 저장한다.

### 6.7 일정 관리

-   일정 목록 조회: AI가 추출한 마감일을 일정 목록에서 조회한다.
-   회의 연결: 일정에서 원본 회의 상세로 이동한다.

## 8. DB 데이터 모델

### 8.1 User

-   역할: 사용자 계정과 프로필 정보 저장
-   주요 필드: id, email, passwordHash, userId, name, phone, company, department, profileUrl, createdAt, updatedAt
-   관계: User 1:N Meeting, User 1:N Category, User 1:N Schedule

### 8.2 Category

-   역할: 사용자가 회의와 일정을 분류하기 위한 카테고리 저장
-   주요 필드: id, userId, name, color, createdAt, updatedAt
-   관계: Category N:1 User, Category 1:N Meeting, Category 1:N Schedule

### 8.3 Meeting

-   역할: 회의 기본 정보와 저장 상태 관리
-   주요 필드: id, userId, categoryId nullable, title, status, memo, startedAt, endedAt, createdAt, updatedAt
-   status 예시: recording, analyzing, analyzed, saved, failed
-   관계: Meeting N:1 User, Meeting N:1 Category nullable, Meeting 1:1 Recording, Meeting 1:1 MeetingAnalysis, Meeting 1:N Schedule

### 8.4 Recording

-   역할: 브라우저 녹음 파일의 메타데이터 저장
-   주요 필드: id, meetingId, fileUrl, fileName, mimeType, durationSec, createdAt
-   관계: Recording N:1 Meeting

### 8.5 MeetingAnalysis

-   역할: AI 분석 결과 저장
-   주요 필드: id, meetingId, transcript, suggestedTitle, summary, mainPointsJson, assigneesJson, dueDate nullable, modelName, createdAt, updatedAt
-   JSON 예시: `mainPointsJson = [{ "content": "다음 배포 범위를 확정한다." }]`
-   관계: MeetingAnalysis N:1 Meeting

### 8.6 Schedule

-   역할: 회의 분석에서 추출된 마감 일정 저장
-   주요 필드: id, userId, meetingId, categoryId nullable, title, assigneeName nullable, dueDate, status, createdAt, updatedAt
-   status 예시: pending, done
-   관계: Schedule N:1 User, Schedule N:1 Meeting, Schedule N:1 Category nullable

### 10.1 디렉토리별 역할

- `docs`: 프로젝트 기획, 규칙, 작업 문서를 관리한다.
- `docs/base/common`: Git 규칙, 프론트엔드 화면 개발 규칙, 검증 실행 규칙, 기술스택 등 공통 개발 기준 문서를 둔다.
- `docs/base/response`: 기획서, 요구사항 정리, 응답 산출물 등 프로젝트 기준 문서를 둔다.
- `docs/tasks`: 기능별 작업 계획과 진행 문서를 둔다. 현재는 `auth`, `dashboard` 작업 문서 영역이 있다.
- `public`: 브라우저에서 직접 제공되는 정적 파일을 둔다.
- `public/icons`: 서비스 아이콘, UI 보조 아이콘 등 정적 아이콘 파일을 둔다.
- `public/images`: 화면에서 사용하는 정적 이미지 파일을 둔다.
- `src`: 애플리케이션 소스 코드의 루트다.
- `src/app`: Next.js App Router 영역이다. 라우트, 페이지, 레이아웃, API 엔드포인트를 둔다.
- `src/app/layout.tsx`: 앱 전체에 적용되는 최상위 레이아웃이다.
- `src/app/page.tsx`: 루트(`/`) 페이지 진입점이다.
- `src/app/api`: 클라이언트가 호출하는 `/api/*` 요청을 받는 Route Handler 영역이다. 실제 비즈니스 로직은 `src/server`로 위임한다.
- `src/app/api/auth`: 회원가입, 로그인, 내 정보 조회 등 인증 API 라우트를 둔다.
- `src/app/api/categories`: 회의/일정 카테고리 API 라우트를 둔다.
- `src/app/api/meetings`: 회의 생성, 목록, 상세, 수정, 삭제, 녹음 저장, AI 분석 요청 관련 API 라우트를 둔다.
- `src/app/api/schedules`: 일정 목록 조회 등 일정 API 라우트를 둔다.
- `src/app/api/users`: 사용자 정보 조회/수정 API 라우트를 둔다.
- `src/app/auth`: 인증 관련 App Router 화면을 그룹화한다.
- `src/app/auth/login`: 현재 로그인 화면 구현 위치다.
- `src/app/login`: 로그인 단독 라우트용 디렉토리다. `/auth/login`과 라우팅 정책을 맞춰 정리 대상이 될 수 있다.
- `src/app/signup`: 회원가입 화면 라우트를 둔다.
- `src/app/dashboard`: 로그인 후 대시보드 화면 라우트를 둔다.
- `src/app/meetings`: 회의 목록 화면과 회의 하위 라우트를 둔다.
- `src/app/meetings/[meetingId]`: 특정 회의 상세 화면 라우트를 둔다.
- `src/app/meetings/record`: 회의 녹음 화면 라우트를 둔다.
- `src/app/schedules`: 일정 목록 화면 라우트를 둔다.
- `src/app/mypage`: 사용자 프로필과 카테고리 관리 화면 라우트를 둔다.
- `src/assets`: 앱 내부에서 import해서 사용하는 정적 리소스와 전역 스타일을 둔다.
- `src/assets/script`: 화면 보조 스크립트나 클라이언트 유틸성 스크립트를 둔다.
- `src/assets/styles`: 전역 스타일과 스타일 토큰을 둔다.
- `src/assets/styles/common`: 기본 CSS, 공통 스타일 인덱스, 디자인 토큰 CSS를 둔다.
- `src/components`: 재사용 가능한 React UI 컴포넌트를 둔다. DB 접근이나 서버 비즈니스 로직은 두지 않는다.
- `src/components/common`: 버튼, 배지, 다이얼로그, 필드, 패널, 탭, 상태 화면 등 공통 UI 컴포넌트를 둔다.
- `src/components/layouts`: 앱 레이아웃 컴포넌트를 둔다.
- `src/components/auth`: 인증 화면 전용 UI 컴포넌트를 둔다.
- `src/components/auth/login`: 로그인 화면 전용 컴포넌트를 둔다.
- `src/components/dashboard`: 대시보드 화면 전용 컴포넌트를 둔다.
- `src/components/meetings`: 회의 목록, 상세, 녹음, 분석 결과 UI 컴포넌트를 둔다.
- `src/components/mypage`: 마이페이지와 카테고리 관리 UI 컴포넌트를 둔다.
- `src/components/schedules`: 일정 목록 UI 컴포넌트를 둔다.
- `src/features`: 기능 단위의 프론트엔드 조합 로직을 둔다. 화면 상태, 액션, 도메인별 클라이언트 로직을 이곳에 모은다.
- `src/features/auth`: 인증 기능의 프론트엔드 로직을 둔다.
- `src/features/dashboard`: 대시보드 기능의 프론트엔드 로직을 둔다.
- `src/features/meetings`: 회의 기능의 프론트엔드 로직을 둔다.
- `src/features/mypage`: 마이페이지 기능의 프론트엔드 로직을 둔다.
- `src/features/schedules`: 일정 기능의 프론트엔드 로직을 둔다.
- `src/services`: 프론트엔드에서 서버 API를 호출하는 클라이언트 함수를 둔다. 예: `fetch("/api/meetings")`.
- `src/stores`: Zustand 등 클라이언트 전역 상태 관리 코드를 둔다.
- `src/types`: 여러 영역에서 공유하는 TypeScript 타입과 인터페이스를 둔다.
- `src/utils`: 프론트엔드와 서버에서 공통으로 사용할 수 있는 순수 유틸리티 함수를 둔다.
- `src/lib`: 특정 라이브러리 래퍼, 공통 헬퍼, 설정성 유틸을 둔다.
- `src/server`: 서버 비즈니스 로직을 둔다. API 라우트에서 호출하는 도메인 서비스를 구현한다.
- `src/server/auth`: 인증, 토큰, 비밀번호 검증 등 인증 도메인 로직을 둔다.
- `src/server/users`: 사용자 프로필 조회/수정 로직을 둔다.
- `src/server/categories`: 카테고리 생성, 조회, 수정, 삭제 로직을 둔다.
- `src/server/meetings`: 회의 생성, 목록, 상세, 수정, 삭제 로직을 둔다.
- `src/server/recordings`: 녹음 파일 메타데이터 저장과 녹음 관련 서버 로직을 둔다.
- `src/server/analysis`: STT, AI 분석 요청, 분석 결과 정제 로직을 둔다.
- `src/server/schedules`: 회의 분석에서 추출된 일정 조회와 상태 관리 로직을 둔다.
- `src/server/common`: 서버 영역에서 공유하는 응답, 에러, 인증 가드, 공통 헬퍼를 둔다.
- `src/database`: DB 연결, Prisma 스키마, 마이그레이션, 시드 등 데이터베이스 관련 코드를 둔다.
- `src/database/prisma`: Prisma 프로젝트 파일을 둔다.
- `src/database/prisma/migrations`: Prisma 마이그레이션 파일을 둔다.
- `next.config.ts`: Next.js 설정 파일이다.
- `package.json`: 프로젝트 스크립트, 의존성, 패키지 메타데이터를 관리한다.
- `tsconfig.json`: TypeScript 컴파일 옵션과 경로 설정을 관리한다.

일반적인 호출 흐름은 `화면(src/app 또는 src/features)` → `서비스(src/services)` → `API 라우트(src/app/api)` → `서버 로직(src/server)` → `DB(src/database)` 순서로 본다.

## 11. 구현 우선순위

1. 로그인, 회원가입, 내 정보 조회
2. 대시보드, 회의 목록, 회의 상세 기본 화면
3. 회의 생성, 브라우저 녹음, 녹음 파일 저장
4. AI 분석 mock API와 분석 결과 팝업
5. 추가 메모 작성 및 회의록 저장
6. 일정 목록, 마이페이지, 카테고리 관리
