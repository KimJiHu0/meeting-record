# 프론트엔드 화면 개발 규칙

## 화면 개발 기본 규칙

-   화면을 제작할 때에는 `/src/app/components/common` 컴포넌트를 우선 적극 활용한다.
-   화면 개발 시에는 최대한 컴포넌트화한다.
-   각 화면별 컴포넌트는 `/src/app/components/[화면]` 내에 정의한다.
-   화면을 제작할 때에는 `docs/base/common/기술스택.md`를 참고한다.
-   화면에 필요한 필드는 `docs/base/response/01. 기획서.md`의 `DB 데이터 모델` 섹션을 참고한다.
-   docs/base/response/01. 기획서.md 파일의 10장 디렉토리 구조, 설명을 기준으로 하여 화면을 제작한다.

## 효율적인 개발 지침

-   작업 전에 요청과 직접 관련된 파일, 컴포넌트, 문서만 좁게 확인한다.
-   요청 범위와 직접 관련 없는 전체 파일 탐색, 전체 문서 정독, 과한 브라우저 검증은 피한다.
-   기존 공통 컴포넌트와 프로젝트 패턴을 재사용해 새 코드 작성량을 줄인다.
-   새 컴포넌트는 실제 중복 제거, 화면 분리, 가독성 개선이 필요한 경우에만 만든다.
-   개발 중에는 구현에 집중하고, 검증 명령은 `docs/base/common/검증_실행_규칙.md`를 따른다.
-   결과 보고는 변경 파일, 핵심 변경 내용, 미실행 검증 여부만 간결하게 전달한다.
