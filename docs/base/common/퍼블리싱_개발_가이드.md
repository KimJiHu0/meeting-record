## 목표

화면 구현이 아닌, 공통 CSS와 공통 컴포넌트 제작이다.
로그인, 대시보드 등 개별 페이지는 만들지 않는다.
공통 퍼블리싱 기반만 구축한다.

## 참고파일

docs/base/response/01. 기획서.md
docs/base/response/02. 전체디자인.md

## 작업 범위

-   전역 CSS 정리
-   디자인 토큰 정의
-   typography, spacing, radius, shadow, color, breakpoint 등등 기준 반영
-   공통 레이아웃 기반 제작
-   Button, Card/Panel, Input/Textarea, Tabs, Badge, Modal/Dialog, Empty/Loading/Error 등등 상태 컴포넌트 제작

## 구현 기준

-   Flow Green 시그니처 컬러와 전체 디자인 문서의 톤앤매너를 반영한다.
-   Next.js와 Tailwind CSS v4 프로젝트 구조에 맞춘다.
-   이후 화면 구현에서 바로 재사용할 수 있도록 props와 variant를 간결하게 설계한다.
-   접근성, disabled, focus-visible, loading 상태를 고려한다.
-   토이 프로젝트이므로 과한 추상화는 피한다.

## 작업 제외

-   개별 페이지 구현
-   API 연동

## 결과물 경로

-   공톨 CSS 위치는 src/assets/styles/common 내부이다.
-   공통 컴포넌트 위치는 src/components/common 내부이다.

## 주의사항

-   프론트앤드 기술스택에 정의된 라이브러리를 사용한다.
-   최대한 간결하고 재사용이 가능한 공통 컴포넌트를 제작한다.
-   반응형에 강한 공통 컴포넌트를 제작한다.
-   공통 컴포넌트 이름은 첫 문자는 대문자로 작성한다. (예시- MeetingDialog, Dialog)
-   공통컴포넌트는 누가봐도 알아볼 수 있는 @docs를 작성한다.
-   전역 CSS는 프론트앤드 개발자가 확인하고 바로 사용할 수 있도록 직관적으로 작성한다.
-   지시한 내용 외에 추가되면 좋을 것들은 제안한다.

마지막으로 가능한 검증 명령을 실행하고 결과를 알려줘.
