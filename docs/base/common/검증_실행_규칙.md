# 검증 실행 규칙

## 기본 원칙

- 일반적인 기능 개발 중에는 요청된 개발 작업만 진행하고 검증 명령은 실행하지 않는다.
- `npm run lint`, `npm run build`, 테스트 실행 등 검증 명령은 커밋/푸시 직전에만 실행한다.
- 사용자가 명시적으로 요청한 경우에는 개발 중에도 검증 명령을 실행할 수 있다.

## 화면 확인 규칙

- 화면 확인용 dev 서버 실행이나 브라우저 검증은 기본적으로 생략한다.
- 사용자가 명시적으로 요청한 경우 또는 커밋/푸시 직전 검증 단계에서만 화면 확인을 진행한다.

## 보고 규칙

- 검증을 생략한 경우 최종 보고에 `검증은 규칙에 따라 실행하지 않음`이라고 명시한다.
