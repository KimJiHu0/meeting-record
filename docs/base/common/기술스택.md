# 토이프로젝트 기술 스택

작성일: 2026-06-19  
문서 목적: 토이프로젝트에서 사용할 Frontend, Backend, Database 기술 스택과 각 기술의 사용 이유를 정의한다.

## 1. Frontend 기술 스택

| 기술 | 역할 | 사용 이유 |
| --- | --- | --- |
| React | 사용자 인터페이스 구성 | 컴포넌트 기반으로 화면을 재사용 가능하게 구성할 수 있고, Next.js와 함께 사용하기 적합하다. |
| Next.js (App Router) | 프론트엔드 애플리케이션 프레임워크 | 라우팅, 서버 컴포넌트, API 연동 구조를 App Router 기반으로 일관되게 구성할 수 있다. |
| TypeScript | 정적 타입 시스템 | 화면, API 응답, 폼 데이터, 상태 값을 타입으로 관리해 개발 중 오류를 줄이고 유지보수성을 높인다. |
| Tailwind CSS | 스타일링 | 유틸리티 클래스 기반으로 빠르게 UI를 구현할 수 있고, 디자인 일관성을 유지하기 쉽다. |
| shadcn/ui | UI 컴포넌트 | 접근성과 확장성을 고려한 컴포넌트를 빠르게 적용할 수 있으며, Tailwind CSS와 잘 맞는다. |
| Zustand | 클라이언트 상태 관리 | 전역 상태를 간단한 구조로 관리할 수 있어 복잡한 보일러플레이트 없이 필요한 상태만 분리하기 좋다. |
| React Hook Form | 폼 상태 관리 | 입력값, 검증 상태, 제출 흐름을 효율적으로 관리할 수 있고 불필요한 리렌더링을 줄이는 데 유리하다. |
| Zod | 스키마 검증 | 폼 입력값과 API 데이터의 유효성을 타입과 함께 검증할 수 있어 TypeScript 기반 개발 흐름과 잘 맞는다. |

## 2. Backend 기술 스택

| 기술 | 역할 | 사용 이유 |
| --- | --- | --- |
| NestJS | 백엔드 애플리케이션 프레임워크 | 모듈, 컨트롤러, 서비스 구조가 명확해 기능별로 코드를 분리하고 확장하기 좋다. |
| Prisma ORM | 데이터베이스 ORM | 데이터베이스 스키마와 쿼리를 타입 안전하게 다룰 수 있고, NestJS와 함께 사용하기 적합하다. |

## 3. Database 기술 스택

| 기술 | 역할 | 사용 이유 |
| --- | --- | --- |
| Supabase | PostgreSQL 기반 데이터베이스 | PostgreSQL을 기반으로 하며, Prisma ORM과 함께 사용할 수 있어 타입 안정성과 관계형 데이터 모델링을 유지하기 좋다. |
